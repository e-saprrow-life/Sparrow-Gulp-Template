=== Файловая структура =========
    app - папка исходника проекта
    build - папка финальной версии проекта


=== JS =========================
    libs.js - файл для всех подключенных библиотек
    script.js - файл основных скриптов, вызова плагинов и прочее
    
    _*.js - файлы которые не обрабатываются, тоесть ведут себя как модули

=== SCSS =======================
    style.scss - для объединения всех стилей проекта


=== GULP NOTES ======================
    gulp.series(...) - выполняет задачи последовательно
    gulp.parallel(...) - выполняет задачи паралельно (одновременно)
    gulp.src() - можно передать несколько путей в масиве:
                 gulp.src(['path1', 'path2']). 
                 '!path/dir/*.min.css' - выберет все файлы кроме тех, что имеют в имени суфикс .min.css
                 Например: '!' + path.build.css + '/*.min.css'

    (!) gulp-clean-css может не работать если папка исходника имеет символ #. Используй правило наименования переменных.

    (i) К команде можно передавать флаги типа --flag, а функция process.argv.includes('--test') позволит записать 
        куда то в настойки был ли этот флаг вызван при вызове програмы
    
    (i) Начало работы с Gulp: 
        - Установить Node js
        - Power Shell (Admin) - Set-ExecutionPolicy RemoteSigned   (Расширяет права) 
        - Power Shell (Admin) - npm install --global gulp-cli      (Установить Gulp глобально) 
        - Установить gulp в проект - npm i


=== ГОРЯЧИЕ КЛАВИШИ VS Code =====================================================================
    - Ctrl + / - закомментировать строку на которой находится курсор
    - Alt + Shift + A   - закомментировать выделенный код
    - Alt + Shift + F   - Форматировать код
    - Ctrl + Shift + W - обернуть в тег
    - Ctrl + Shift + B - Скопировать БЄМ структуру селекотров
    - Ctrl + B         - Показать / скрыть боковую панель
